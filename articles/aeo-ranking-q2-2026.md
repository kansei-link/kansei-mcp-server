## 5つの注目ポイント

### 1. Slackが「エージェント経済のインフラ」に

Slack MCP（AAA、35レシピ）は他サービスとの連携レシピ数で圧倒的1位。エージェントがfreeeで請求書を作成してもSmartHRで入社処理をしても、通知先はSlack。**AIエージェント時代の「通知ハブ」としての地位を確立**している。

### 2. freee vs MoneyForward：会計カテゴリの明暗

両社ともAAグレードだが、内訳に差がある。freeeはレシピ数19で他サービスとの接続パターンが豊富。MoneyForwardは成功率93%と安定性で勝る。**エージェントにとっての「使いやすさ」はAPIの質だけでなく、エコシステムの厚みで決まる。**

### 3. SmartHR：需要はあるのに成功率40%の課題

SmartHRは12件のレシピで需要の高さを示す一方、エージェント成功率は40%にとどまる。人事データは構造が複雑で、エージェントがつまずきやすい。**API設計の改善とエラーメッセージの充実が、AEOスコア向上の鍵**となる。

### 4. 決済カテゴリは「未開の地」

決済カテゴリの平均AEOスコアは0.44と全18カテゴリ中最低。Verified認定を受けたサービスはゼロ。Stripe Japan（BBB）がリードしているが、Square、LINE Pay、PAY.JPはいずれもBB〜Bにとどまる。**エージェントが決済処理を自動化できる日はまだ遠い。**

### 5. 法務・契約：MCP未対応がボトルネック

CloudSign、DocuSign Japan、freeeサインの上位3社はすべてBBBグレードで横並び。いずれもAPIは持つがMCPサーバーは未提供。**MCP対応を先行した1社がカテゴリリーダーとなるチャンスがある。**

## AEOスコア評価方法

### スコア構成（0.00〜1.00）

**基盤スコア（Base Score）：**

| 接続方式 | Base Score | 解説 |
|----------|-----------|------|
| Official MCP Server | 0.50 | MCP公式サーバー提供 |
| Third-party MCP | 0.40 | コミュニティMCPサーバー |
| API Only | 0.30 | REST/GraphQL API提供 |
| No API | 0.10 | プログラマティック接続手段なし |

**ボーナス加点（各+0.10、最大+0.50）：**

| ボーナス | 条件 |
|---------|------|
| APIドキュメント | エンドポイント一覧、認証方法が文書化 |
| 認証ガイド | OAuth/APIキー取得手順が公開 |
| カテゴリ特化 | 専門領域に集中したサービス |
| エージェント利用実績 | 1回以上のエージェント経由利用 |
| 高成功率 | エージェント成功率80%以上（3回以上） |

### Agent Readyステータス

| ステータス | 判定基準 |
|-----------|---------|
| Verified | MCP/APIあり + エージェント3回以上利用 + 成功率80%以上 |
| Connectable | MCP/APIあり、未検証または成功率が低い |
| Info Only | プログラマティック接続手段なし |

### データソース

- **エージェント利用データ：** KanseiLink MCP Intelligence Layer経由のリアルタイム利用テレメトリ
- **API監査：** 公式ドキュメント、認証フロー、エンドポイント可用性の手動検証
- **MCPレジストリ：** modelcontextprotocol.io 公式レジストリスキャン
- **ワークフロー検証：** 100件のクロスサービス連携パターンの動作確認
