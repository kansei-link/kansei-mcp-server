## はじめに

2026年の日本において、経理業務のAI自動化は「検討フェーズ」から「導入・運用フェーズ」へと完全に移行した。この変化を牽引しているのが、クラウド会計ソフトとAIエージェントを結ぶMCP（Model Context Protocol）の普及である。

freee会計は、日本のSaaS会計市場において中小企業から上場準備企業まで幅広い層に浸透しているサービスだ。2026年4月時点で、freeeは公式MCPサーバーを提供しており、AIエージェントからの直接的な経理業務自動化が実現可能な状態にある。本記事では、freee APIの特性、MCP連携の具体的手順、そしてエージェント運用者が押さえるべきポイントを網羅的に解説する。

## freee MCPサーバーの基本構成と接続方法

### 公式MCPサーバーの概要

freeeは公式MCPサーバーを`npx @freee/mcp-server`というエンドポイントで提供している。Namespaceは`io.github.freee/accounting`であり、Remote MCP接続にも対応している。

このMCPサーバーの特筆すべき点は、freeeが展開する5つのビジネスドメイン——会計（Accounting）、請求書（Invoicing）、人事労務（HR）、給与計算（Payroll）、勤怠管理（Time Tracking）——を1つのサーバーで統合的に扱える設計になっていることだ。

### 認証方式：OAuth 2.0+PKCE

freee MCPの認証はOAuth 2.0+PKCEを採用している。これはAIエージェントからの接続においてセキュリティと利便性を両立させる設計である。

PKCEフローの採用により、クライアントシークレットをエージェント環境に保存する必要がなくなり、サーバーレス環境やエッジコンピューティング環境での運用が容易になっている。エージェント運用者は、アクセストークンの発行・リフレッシュフローを自動化するラッパー処理を組み込むことで、長期運用に耐えうる構成を実現できる。

### 接続コマンドの基本

```
npx @freee/mcp-server
```

上記コマンドを実行することで、ローカル環境またはクラウド環境にMCPサーバーが起動する。公式ドキュメントは[freee Developers](https://developer.freee.co.jp/docs/accounting/reference)で確認できるため、詳細なパラメータ設定については要参照だ。

## レートリミット設計とエージェント運用の注意点

### リクエスト制限の具体値

freee APIのレートリミットは以下のように設定されている。

| 対象 | 制限値 | 備考 |
|------|--------|------|
| 一般エンドポイント | 3,600リクエスト/時（アクセストークンあたり） | 1秒あたり約1リクエストの平均ペース |
| Wallets/dealsエンドポイント | 300リクエスト/5分 | 口座・取引系の高頻度アクセスを制限 |
| 超過時のレスポンス | HTTP 429 | rate-limitヘッダーで残量確認可能 |

### エージェント設計への影響

AIエージェントが経理業務を自動化する際、最も陥りやすい落とし穴が「レートリミット超過による処理停止」である。特に月次決算時や年度末の繁忙期には、仕訳登録・請求書発行・取引明細取得が集中するため、リクエスト設計の工夫が必要だ。

推奨されるアプローチとしては、以下が挙げられる。

1. **バッチ処理の最適化**：複数の仕訳をまとめて登録できるバルクAPIがある場合は積極的に活用する
2. **キャッシュ戦略**：勘定科目マスタや取引先マスタなど、変更頻度の低いデータはローカルキャッシュを活用
3. **リトライ戦略**：HTTP 429を受け取った際、rate-limitヘッダーのリセット時刻を参照して適切なバックオフを実装
4. **ピークシフト**：日次バッチ処理を深夜帯に集約し、日中のリアルタイム処理枠を確保

## AIエージェントによる経理自動化レシピ

freee MCPを活用した経理自動化には、すでに複数の実運用レシピが確立されている。KanseiLINKのサービスDBに登録されている代表的なレシピを紹介する。

### 請求書処理の自動化（invoice-processing）

請求書の作成から会計データへの同期までを一気通貫で処理するレシピ。受領した請求書PDFをOCR処理し、抽出されたデータをfreee会計に仕訳として登録する流れが典型的だ。

### 契約締結→仕訳登録の連携（contract-to-accounting）

電子契約サービスで締結された契約内容を起点に、売上計上・前受金処理などの会計仕訳を自動生成する。特に日本の商習慣では契約書と請求書のタイミングにズレが生じることが多く、このレシピによって計上漏れを防止できる。

### EC売上の自動同期（ec-accounting-sync）

ECプラットフォームの売上データをfreee会計に自動で取り込む。売上・手数料・振込金額の3点を正確に仕訳するには、ECプラットフォーム側のAPIとfreee APIを同時に扱うエージェント設計が求められる。

### kintoneとの顧客マスタ同期（kintone-freee-client-sync）

kintoneで管理している顧客マスタをfreee会計の取引先マスタと同期するレシピ。マスタの二重管理を解消し、請求先情報の整合性を担保する。

### Stripe決済の自動仕訳（stripe-freee-payment-sync）

Stripeで処理された決済情報をfreeeに自動で仕訳登録する。サブスクリプションビジネスにおいては、月次の売上計上・前受金消込・手数料計上を自動化できる点が大きなメリットとなる。

## 日本SaaSとしてのfreee API設計の特徴

### 日本特有の業務慣習への対応

freee APIは日本の経理業務に最適化された設計がなされている。消費税のインボイス制度対応、電子帳簿保存法への準拠、年末調整処理といった、日本固有の法制度・慣習への対応が組み込まれている。

AIエージェントから見た場合、この「日本ローカライズの深さ」は両刃の剣でもある。海外SaaSのAPIと比較してパラメータが複雑になる傾向があり、エージェントのプロンプト設計においては日本の税務・会計知識を適切に組み込む必要がある。

### ドキュメントとエラーメッセージの品質

freee Developersのドキュメントは日本語で整備されており、エンドポイントごとのリクエスト/レスポンス例も充実している。エラーメッセージも日本語で返却されるため、AIエージェントがエラー内容を解釈して自己修復処理を行う際の精度が向上する。

一般的に、英語圏SaaSのAPIと比較した場合、日本のSaaS APIはエラーメッセージの丁寧さにばらつきがあることが多い。freeeはこの点において相対的に高い品質を維持していると評価できる。

### Trust Scoreの意味

KanseiLINKのサービスDBにおいて、freeeのTrust Scoreは0.8と記録されている。これはAPI品質、ドキュメント整備度、MCP対応状況などを総合評価したスコアであり、AIエージェントからの利用に適したサービスとして位置づけられている。

## 導入ステップと実装チェックリスト

freee MCPを導入する際の標準的なステップを整理する。

1. **freee Developersへの登録**：開発者アカウントを作成し、OAuth 2.0のクライアントID/シークレットを取得
2. **MCPサーバーのセットアップ**：`npx @freee/mcp-server`でローカル起動を確認
3. **OAuth認証フローの実装**：PKCE対応のトークン取得・リフレッシュ処理を組み込み
4. **テスト環境での疎通確認**：サンドボックス環境またはテスト事業所で基本的なCRUD操作を検証
5. **本番環境への移行**：レートリミットを考慮したバッチ設計・エラーハンドリングを実装
6. **運用監視の整備**：API呼び出し回数、エラー率、処理時間のモニタリングを構築

## FAQ

**Q1. freee MCPを使うにはどのような開発環境が必要ですか？**
A. Node.js環境があれば、`npx @freee/mcp-server`コマンドで即座にMCPサーバーを起動できます。OAuth 2.0+PKCE認証を行うため、freee Developersでクライアント登録を行い、アクセストークンを取得する処理を実装する必要があります。Remote MCP接続にも対応しているため、クラウド環境やサーバーレス環境での運用も可能です。詳細なセットアップ手順は公式ドキュメントを参照してください。

**Q2. レートリミットを超過した場合、どのように対処すべきですか？**
A. HTTP 429ステータスが返却された際、レスポンスヘッダーにrate-limit情報が含まれています。この情報を参照してリセット時刻までバックオフする処理を実装することが推奨されます。また、繁忙期（月次決算・年度末）にはリクエストが集中するため、バッチ処理を深夜帯にシフトする、キャッシュを活用してAPIコール数を削減するなどの工夫が有効です。

**Q3. freee MCPは会計以外のドメイン（HR・給与など）にも対応していますか？**
A. はい、freeeの公式MCPサーバーは会計（Accounting）、請求書（Invoicing）、人事労務（HR）、給与計算（Payroll）、勤怠管理（Time Tracking）の5つのビジネスドメインを1つのサーバーで統合的に扱える設計となっています。これにより、経理部門だけでなくバックオフィス全体のAI自動化をシームレスに構築できる点が大きな特徴です。
