「MCP対応」チェックボックスの罠

AIエージェントの普及が加速する2026年春、「MCP対応済み」という表示が企業SaaS調達の新しい評価項目になりつつある。ITベンダーはプレスリリースでMCPサーバーの提供を発表し、調達担当者は「MCP対応？」というチェックボックスに印をつけて次の項目に移る。

しかしKanseiLinkが実測した225+サービスのAgent Readinessデータは、この問いかけが根本的に間違っていることを示している。

「MCP対応？」は間違った質問だ。正しい質問は「どのtierのMCP対応で、実際の成功率はいくつか？」だ。

KanseiLinkの3段階Agent Readiness評価

KanseiLinkはMCPサービスのAgent Readinessを3段階で評価している。この分類はAEOグレード（AAA/AA/A/BBB等）とは独立した指標だ。

🟢 verified — 実戦検証済み

実際のエージェント呼び出しデータが十分に蓄積され、成功率80%以上が確認されているサービス。主要なエラーパターンと既知のワークアラウンドが文書化されており、本番環境でのエージェント統合に推奨できる状態。

KanseiLink 2026年4月: 6サービス該当 | 成功率レンジ: 83〜94%
🟡 connectable — 接続可能だが未実証

公式MCPサーバーまたは十分なAPIが存在し、技術的に接続可能だが、実戦データが不十分または成功率が低いサービス。「接続できる」と「エージェントが問題なく使える」は別の話であることを意味する。

KanseiLink 2026年4月: 多数 | 成功率レンジ: 13〜100%（データポイント数に大きく左右される）
⚪ info_only — 情報参照のみ

MCPサーバーが存在せず、APIも限定的なサービス。KanseiLinkは基本情報（サービス概要、カテゴリ、価格帯など）を提供できるが、エージェントが実際に操作できる状態ではない。

調達検討段階のサービスや、エージェント連携が将来計画のサービスが該当
重要: verifiedは静的ではない

Agent Readiness tierは変動する。成功率が向上し、データが蓄積されれば、connectableからverifiedに昇格する。逆にAPIの仕様変更や成功率低下があればverifiedからconnectableに降格することもある。KanseiLinkのデータは継続的に更新されている。

verifiedの実態: 安定した高成功率クラブ

2026年4月時点でKanseiLinkがverified評価を付与しているサービスは6つ、すべてAAA評価を持つ。その成功率は驚くほど一定している。

verifiedサービスの平均成功率は89%。最低でも83%（Notion）、最高は94%（Shopify Japan）。分散が小さく、本番環境のエージェントワークフローに組み込める信頼性を持つ。

verifiedの共通特徴

全6サービスはAAA AEOグレードを持ち、公式npmパッケージとして公開されたMCPサーバーを持ち、十分なデータサンプル数が蓄積されている。「公式MCPサーバー × 継続的なエージェント利用 × 品質管理」の3要素がverifiedへの道だ。

connectableの実態: 大荒れの13〜100%

connectableは「公式MCPが存在するが未実証」というカテゴリだが、その内部は均質ではない。KanseiLinkのデータが示すconnectable内の分布は衝撃的だ。

同じconectable tierの中でも、Microsoft TeamsとZapierの間には87ポイントもの成功率差がある。「connectableだから大丈夫」という判断は、13%の地雷を踏む可能性と100%の優等生を同列に扱っていることになる。

⚠️ 「100%」のconnectableに注意

Microsoft Teams、LINE Messaging、PostgreSQL MCPの成功率は100%だが、これはデータポイントが少ないことによる可能性が高い。少数の呼び出しで全て成功した場合に100%になるが、大規模利用時に同じ成功率を維持するかは未検証。これらのサービスがverifiedでなくconnectableに留まっている理由はここにある。

グレード × Readiness マトリクス: 驚くべきズレ

AEOグレードとAgent Readiness tierの関係を見ると、重要なことに気づく。

AEOグレードとAgent Readiness tierは別物だ。

AEOグレード（AAA/AA/A/BBB）はAPIの設計品質、ドキュメント充実度、MCPサーバーの実装完成度、セキュリティ対応などの総合評価だ。一方、Agent Readiness tierは「実際にエージェントが使った場合の成功率と信頼性」の評価だ。

例えば、A評価のZapierはconnectableだが成功率13%——設計の品質と実際の運用結果が乖離しているケースだ。逆にA評価のMicrosoft TeamsはconnectableながらデータPoC段階では100%を示している。

調達判断の実践的解釈

AAA + verified = 本番推奨。エンタープライズのコアワークフローに組み込める。
AA/A + connectable（高n・高成功率） = 慎重に採用可。PoC後の本番移行を検討。
AA/A + connectable（低n または 低成功率） = パイロットのみ。ベンダーへのAEO改善要求を推奨。
A + connectable（Zapier型） = 現時点での本番採用は非推奨。

エンタープライズ調達の新ガイド: 5つの問い

「MCP対応ですか？」という質問を卒業し、以下の5つを問うことを推奨する。

  1. Agent Readiness tierは何ですか？ (verified / connectable / info_only)
    KanseiLinkのAEO評価レポートか、ベンダーに直接確認する。「MCP対応」という回答はconnectable以上を意味するが、それだけでは不十分。
  2. 実際の成功率はいくつで、サンプル数はいくつですか？
    n=9の100%とn=113の91%では信頼性が全く異なる。最低n=50以上で成功率80%以上がエンタープライズ本番導入の目安。
  3. 主要なエラーパターンとワークアラウンドは文書化されていますか？
    verifiedサービスは既知の問題と解決策が文書化されている。これがないサービスは、本番トラブル時の対応コストが高くなる。
  4. 直近30日の成功率と信頼性トレンドはどうですか？
    APIの仕様変更や機能追加によって成功率が急落することがある。モメンタムの確認が重要。
  5. ベンダーはAEOスコアの継続改善にコミットしていますか？
    verifiedを維持・向上させる姿勢があるベンダーかどうか。KanseiLinkへの登録と継続的なデータ提供が一つの指標になる。

SaaSベンダーへの示唆: verifiedを取りに行く

2026年以降、エンタープライズの調達担当者はAEOグレードとAgent Readiness tierの両方を確認するようになる。「MCP対応済み」という表示だけでは差別化できない時代が来ている。

verifiedを獲得するためのロードマップ：

  • 公式MCPサーバーをnpmパッケージとして公開し、ドキュメントを充実させる
  • KanseiLinkへの登録と定期的なAEOスコア監査の実施
  • エージェント呼び出しの実測データを蓄積するためのテスト環境提供
  • 主要エラーパターンの特定と公式ワークアラウンドドキュメントの作成
  • MCPツールのdescriptionフィールドを日本語エージェントの意図パターンに最適化

FAQ

verifiedとconnectableの違いは何ですか？

verifiedは成功率80%以上かつ十分なデータサンプル数（n≥50相当）が蓄積されたサービスです。connectableは公式MCPが存在するが実戦データが不十分または成功率が基準未満のサービスです。2026年4月時点でverified 6サービスの成功率は83〜94%に集中、connectableは13〜100%と大きな幅があります。

「MCP対応」と表示されているサービスは信頼できますか？

「MCP対応」という表示だけでは信頼性の判断はできません。ZapierはA評価でconnectableですが成功率13%、Microsoft TeamsはA評価でconnectableですが（少ないデータで）100%です。必ず成功率とサンプル数を確認してください。

verifiedになるにはどうすればいいですか？

必要条件: ①公式MCPサーバーの提供、②成功率80%以上、③n≥50相当のデータ蓄積です。KanseiLink AEO改善プログラムを通じてデータ収集を加速し、verifiedへの道を計画的に進める方法があります。

データ開示・免責事項

本記事のデータはKanseiLink MCPシステムが収集したエージェント呼び出しログに基づきます。サービスによりサンプル数に差があり、特に少数データのサービス（Teams、LINE Messaging等）の数値はデータ蓄積により変動します。Agent Readiness tierは定期的に更新され、現在の評価と異なる場合があります。最新データはKanseiLink MCPサーバー経由でご確認ください。
