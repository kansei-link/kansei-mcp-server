評価フレームワーク — Agent Readinessの5領域

Agent Readinessは単一の指標ではありません。5つの評価領域で定義し、それぞれの証拠を「公開情報 → 自社実測 → 複数AIの実利用データ」の順に硬くしていきます。すべてのスコアにデータ出所ラベル（公開シグナル／プローブ実測／実利用テレメトリ）を付与し、推定を実測として表示することはありません。

格付けは何を測定するか

本データベースの格付けは、SaaSサービスがAIエージェントにどれだけ対応しているかを評価します。ルールベースの方法論（LLMの主観判断は不使用）により、MCP対応状況、API公開、ドキュメント、認証方式の明確さに基づいて格付けを付与し、自社プローブによる接続実測で補強します。

11,000+のSaaS/MCPサービスを23のカテゴリにわたって評価。うち2,257件は自社プローブ（JSON-RPCハンドシェイク）で接続を実測済みです。個社の接続成功率の数値は、独立した外部エージェントのテレメトリが十分な規模に達するまで公開しません。

本データベースで用いるAEO（Agent Engine Optimization）スコアは、Agent Readiness 5領域のうち、主に AI Access と AI Discoverability を測る構成要素です。

格付けの活用方法

AIエージェント、SaaS企業、インテグレーターがKanseiLinkの格付けを活用する6つの方法

サービスの発見

エージェントが目的に最適なSaaSを自動的に発見。Verified認定サービスを優先選択。

リスクの評価

接続先SaaSの接続方式・検証状況・既知の落とし穴を事前に評価し、失敗リスクを最小化。

ワークフロー構築

200件のレシピから最適な連携パターンを取得（12件はMCP検証済みツール名付き）。マルチサービス自動化を構築。

対応力のベンチマーク

SaaS企業が自社のエージェント対応力を業界内で比較。改善ポイントを特定。

改善の推進

格付け改善に必要な具体的アクションを提案。MCP対応、API設計、ドキュメント整備。

変更の監視

API変更、MCP更新、障害情報をリアルタイム追跡。エージェントのワークフローを保護。
