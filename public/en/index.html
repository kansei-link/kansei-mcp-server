The framework — five domains of Agent Readiness

Agent Readiness is not a single metric. We define it as five domains, each hardened progressively: public facts → first-party probes → real usage data from multiple AI agents. Every score carries a data-source label (public signal / probe / usage telemetry) — estimates are never presented as measurements.

What do KanseiLink ratings measure?

Our service ratings assess how ready a SaaS service is for AI agents. Using a rules-based methodology — no LLM judgment — we assign a letter rating based on MCP availability, API openness, documentation, and authentication clarity, supplemented by first-party connectivity probes.

We evaluate 11,000+ SaaS/MCP services across 23 categories; 2,257 have been probe-tested by our own scanner via real JSON-RPC handshakes. Per-service success-rate figures are not published until independent third-party agent telemetry reaches sufficient scale.

The AEO (Agent Engine Optimization) Score used in this database measures primarily the AI Access and AI Discoverability domains of the Agent Readiness framework.
