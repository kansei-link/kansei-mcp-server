The State of AEO in the CRM Category

AI agents searching deal history, updating contact records, and autonomously advancing sales pipelines — this is no longer a thought experiment. In Q2 2026, KanseiLink's agent telemetry shows a sharp rise in CRM category access attempts, making it one of the most active categories across all 18 tracked segments.

Yet there is a substantial gap between "the CRM that dominates enterprise deals" and "the CRM that AI agents can reliably use." This report cross-references real agent data from four major CRM platforms operating in the Japanese market — Sansan, HubSpot, SALES GO, and Salesforce — and assigns AEO grades based on observed performance.

Data Context

Salesforce: 42 records, Sansan: 35 records, HubSpot: 3 records, SALES GO: 1 record. This distribution directly reflects the pace of adoption in the agent developer community. Trust scores incorporate sample volume, success rate, latency, and MCP server status.

Business card management and CRM platform Sansan launched its official MCP server in November 2025 and has begun accumulating enterprise deployment experience, including a Microsoft 365 Copilot integration trial at Sumitomo Corporation. KanseiLink data shows an average latency of 173ms — among the fastest in this comparison — with stable performance for contact tracking, business card OCR, and meeting history retrieval.

A 60% success rate may look modest at first glance, but the error breakdown tells a more nuanced story. The primary failure drivers are API errors (9 cases) and search misses (5 cases). The search misses stem from discoverability gaps: natural language queries like "show me a list of contacts from this account" often fail to surface Sansan in the top results. As query optimization and agent-facing documentation improve, an upgrade to AAA is within reach.

Tips for Connecting Agents to Sansan

  • Use Sansan-specific keywords in queries: "business card," "meishi," "contact management"
  • Structure prompts around Sansan's three-axis data model: companies, contacts, and meeting history
  • Authentication is API key-based — straightforward to implement in agent workflows

Salesforce dominates Japan's enterprise CRM market, but its AI agent performance data tells a sobering story. With no official MCP server (Salesforce restricts MCP to its internal Agentforce framework), agents must rely on third-party implementations — an architectural liability that shows up clearly in the numbers.

Across 42 agent runs, Salesforce records a 43% success rate and 474ms average latency — the weakest across all four platforms in this comparison. Error analysis reveals API errors (12 cases) and search misses (11 cases) in near-equal proportion. Common queries like "auto-log support inquiries to CRM" or "manage deals in SFA" consistently fail to surface Salesforce in top search results, compounding the core reliability issue.

Known workarounds exist: adding LIMIT 200 and date filters to SOQL queries reduces timeout risk, and batching large requests improves throughput. But the absence of official MCP means every agent implementation bears an ongoing engineering maintenance cost that simply does not exist with official-MCP platforms.

Key Considerations for Salesforce Agent Integration

  • Always append LIMIT clauses and date filters to SOQL queries to prevent timeouts
  • Include "Salesforce" or "SFDC" explicitly in search queries to improve discovery
  • Implement robust error handling and retry logic — standard practice until official MCP arrives

The most striking pattern in this dataset is structural: Japan-origin platforms (Sansan, SALES GO) have outpaced the global CRM leader (Salesforce) in official MCP adoption. Whether Salesforce eventually opens MCP access hinges on the competitive dynamics between its proprietary Agentforce strategy and the broader open-protocol ecosystem. How this tension resolves will shape the CRM AEO landscape through 2027 and beyond.

Frequently Asked Questions

QWhich CRM is better for AI agent integration in Japan — Salesforce or Sansan?
It depends on use case. For Japanese business card and contact management workflows, Sansan (AA, 173ms, official MCP) is the stronger choice today. Salesforce offers more comprehensive global SFA functionality, but its BB grade (43% success, 474ms, no official MCP) means additional engineering is required for reliable agent workflows.
QWhat is SALES GO and why does it matter?
SALES GO launched Japan's first official MCP-enabled SFA server in October 2025, offering pipeline management and sales forecasting at roughly one-quarter of Salesforce's cost. With only 1 telemetry record, it holds BBB today — but its native-MCP architecture makes it the domestic contender most aligned with where the agent economy is heading.
QWhat is the difference between Salesforce Agentforce and MCP?
Agentforce is Salesforce's proprietary AI agent framework, designed to operate within its own ecosystem. MCP (Model Context Protocol) is an open standard from Anthropic that enables any external agent to connect to any tool via a common interface. Salesforce currently restricts MCP to Agentforce's internal use, meaning third-party agents must use unofficial servers with lower reliability.
